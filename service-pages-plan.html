Background

What was already done on water-thermal-insulation-system.html

water-thermal-insulation-system.html was used as the reference page. Everything below is based on what was built and refined there. The same two phases will be applied to all 13 remaining pages.

Phase A — Layout
  1. Clean 2-column layout
    Content column (col-8) + sticky sidebar (col-4). Replaced the original messy nested sections and heavy inline styles.
  2. Structured content sections
    Each section follows the pattern: h2 heading → image → list or steps. Uses 4 section types: definition, features list, numbered steps, note box.
  3. Sidebar components
    WhatsApp CTA button + service navigation links (with active state) + PDF download + contact info (phone & email).
  4. Edit-friendly comments
    Every editable element has an <!-- Edit: what to change --> comment directly above it. No need to search the code.
  5. CSS moved to main.css
    All 6 component classes (.section-heading, .section-img, .features-list, .steps-list, .note-box, .cta-whatsapp) are in assets/css/main.css under the "SERVICE PAGE COMPONENTS" block, fully documented with usage examples.
Phase B — SEO
  1. Stronger meta description
    ~150 characters, includes primary keywords naturally, ends with a call-to-action.
  2. Canonical URL
    <link rel="canonical" href="https://tabqat.com.sa/page.html"> — tells Google the authoritative URL for each page.
  3. Meta robots
    <meta name="robots" content="index, follow"> — explicitly tells search engines to index the page.
  4. Open Graph tags
    og:title, og:description, og:image, og:url, og:type, og:locale, og:site_name — controls the preview when a link is shared on WhatsApp or social media.
  5. Twitter Card tags
    twitter:card, twitter:title, twitter:description, twitter:image — controls the preview when shared on X (Twitter).
  6. JSON-LD: Service schema
    Structured data that tells Google the page is about a specific service, who provides it, where, and how to contact them. Enables rich results in search.
  7. JSON-LD: BreadcrumbList schema
    Structured data for the breadcrumb trail. Google shows this directly in search results under the page title.
  8. Heading hierarchy fix
    Changed section headings from h3 → h2 so the page follows a proper h1 → h2 flow. Search engines use this hierarchy to understand page structure.
  9. Descriptive image alt texts
    Changed generic alt values (e.g. "image") to keyword-rich descriptions (e.g. "عزل الأسطح بنظام رش البولي يوريثان بالرياض"). Google indexes alt text for image search.
